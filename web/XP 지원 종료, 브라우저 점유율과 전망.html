들어가기

2014년 4월 8일 Widnows XP 지원 종료 예정 입니다. 다양한 점이 변화되겠지만, 웹 개발자로서 관심이 가는 점은 HTML5 및 CSS3를 사용하는 항상 걸림돌이 되어왔던 IE 8 (Internet Explorer 8) 점유률이 어떻게 변할 것인가 입니다. 이에 현재 데스크탑 브라우저 점유률 확인 및 전망을 해보았습니다. 

IE 8, IE 9

 HTML 5 및 CSS 3과 관련해서 주목해야하는 브라우저는 역시 IE 시리즈 입니다. 왜냐하면 크롬과 파이어폭스는 자동 업데이트를 통해서 대부분 최신 브라우저로 업데이트가 되기 때문입니다.

[##_1N|cfile30.uf@2178694E532EFF1704E524.png|width="606" height="402" alt="브라우저 별 HTML5 지원" filename="browserHtml5Support.png" filemime="image/jpeg" style="text-align: center;"|브라우저 별 HTML5 지원_##]

 위 표( html5test.com )에서 알수 있듯이  점유율(아래 표 참고) 5% 이상을 가지면서 사용되는 데스크탑 브라우저 중에 HTML5 사용에 문제가 되는 브라우저가 IE 8 과 IE 9 입니다. CSS3( CSS3 Browser Support Reference )도 HTML5와 비슷하게  IE 8  과  IE 9 가 문제가 되는 상황입니다. (그렇다고 IE 10, IE 11이 좋은 것은 아닙니다. 그나마 좋다는 것입니다.)

 이렇게 IE 8, IE 9가 문제가 되는 상황에서 'Windows XP 지원 종료'는 긍정적으로 생각하면 Windows XP 사용자가 Windows 7 또는 8으로 OS를 변경하는 것이고, IE 8 및 IE 9 사용자들이 IE 11 사용자로 전환되는 것으로 이어지기 때문입니다.

 개발자의 시각에서 보면 다양한 javascript, css 라이브러리를 IE 8, IE 9에 고통받지 않으면서 사용할 수 있는 여건이 되는 것 입니다. (생각만 해도 흥분되네요. :) ) 좀 더 자세한 이야기는 테스트톱 웹 브라우저 및 OS 점유율를 통해서 이야기 해보겠습니다.

 2014년 3월 현재 국내 브라우저 점유률은 IE 8(30.41%) 이 압도적인 1위이 가운데 크롬(20.77%) 이 뒤따르고 있고, IE 9(17.31%) , IE 10(14.37%) , IE 11(13.76%)  순 입니다.

 지난 1년간 추이를 보면 IE 8과 크롬은 큰 변화가 없지만, 다행히도 MS의 Windows Update와 IE 자동 업데이트를 통해서  IE 9 , IE 10 , IE 11 사이에서 변화를 확인할 수 있습니다. 1년 전에는 IE 9 를 IE 10 로 대체가 되는 모습을 보이다가 현상을 유지하였고, 현재는 IE 10 이 IE 11 로 천천히 대체되는 것을 볼 수 있습니다.

 IE 9가 IE 10으로 대체가 진행되다가 일정 수준을 유지하는 것을 볼 수 있는데, 업데이트 방지 설정을 한 사용자들로 예상을 해 볼 수 있습니다. IE 8 수준은 아니지만 저 17%는 꽤나 오래 지속될 수도 있다는 불길한 예감이 드는 순간입니다. 이와 비슷하게 현재 IE 10도 IE 11로 대체되는 듯한 모습을 볼 수 있는데 일정 수준까지 떨어지고 멈출 것으로 예상해 볼 수 있습니다.

운영체제의 경우는 Windows 7(75.17%) 의 성공, Windows 8(4.43%) 의 실패, Windows XP(15.66%) 의 저력으로 인해서 Windows 7과 Windows XP가 현재 가장 중요한 운영체제인 것을 확인할 수 있습니다.

 1년간 추이를 보면 좀 더 흥이로운 사실을 확인할 수 있습니다. Windows XP 점유률( -18.29% )이 떨어지면서, 해당 점유율을 Windows 7( +18.43% )이 대부분 흡수를 했다는 점입니다. 안타깝게도 Windows 8( +0.67% )은 거의 상승하지 않았다는 점입니다.

 이로 유추해 볼 수 있는 점은 2가지 정도인 것 같습니다.

  1. Windows XP의 점유율은 지속적으로 낮아지고 있고, XP 지원 종료에 따라서 좀 더 빠른 속도로 점유율이 하락할 것이다.
  2. Windows XP 사용자들이 대부분 Window 7으로 업그레이드 할 확률이 높다.

추가 고려사항

 좀 더 고려할 사항들이 몇 가지 있습니다. 우선 Windows 만의 이슈입니다.

  1. Windows XP는 IE8까지만 지원하고, 현재 점유율상 대부분의 XP 사용자는 IE 8를 사용하고 있다는 점입니다.
  2. WIndows 7의 기본 브라우저는 IE 8 입니다.
  3. Windows 7는 IE 11까지 설치할 수 있습니다.
  4. Windows 8의 기본 브라우저는 IE 10 입니다.

다음은 외부적인 요소입니다.

  1. 최초에 IE9, IE10이 출시되었을 때는 은행 및 관공서 홈페이지가 정상적으로 지원하지 않아서 IE 8 및 IE 9를 유지하는 사용자가 많았다.
  2. 현재는 많은 사이트들이 IE9 ,IE10, IE11 을 지원하고 있다.
  3. Windows XP 지원이 종료되지만, 별도의 비용을 들여서 Windows를 업데이트 할 지는 의문입니다. (물론 어둠의 경로를 이용해서 많은 분들이 설치할 것이라고 예상을 해 봅시다.)

예 상

 Windows XP 사용자들이 지금보다는 빠른 속도로 Windows 7 또는 Windows 8로 업데이트를 하지 않을까 생각합니다. 현재까지의 추세를 보면 Windows 7로 많이 업데이트를 할 것 같고, 최초의 브라우저는 여전히 IE 8이 되지 않을까 생각합니다.

 하지만 최초 Windows 업데이트를 통해서 대부분 IE 11로 업데이트로 업데이트 되기 때문에 실제로는 IE 8 점유율을 줄이고, IE 11의 점유율을 높이는 결과로 나타나지 않을까 생각됩니다.

 또한 시대 흐름이 HTML5와 CSS3를 이용한 웹 개발이 지속적으로 관심을 받고 있고, IE8 지원을 위해서 많은 자원이 투입되고 있기 때문에 예전 IE 6의 점유율을 낮추기 위해서 진행했던 캠페인과 비슷한 노력들이 진행될 것으로 생각됩니다.  Gmail 같은 경우는 IE 8로 접속하면 아래와 같은 Alert을 표시하고 IE 업데이트 또는 크롬 사용을 권장하고 있습니다.

 물론 웹 사이트를 만드는 기업 및 관공서들이 무시할 수 있는 수준인 5% 이하로 점유율이 떨어지는 것은 시간이 필요할 것으로 생각됩니다.

결 론

[##_1N|cfile10.uf@251C7343532E70C20F9236.png|width="670" height="145" filename="browsers.png" filemime="image/jpeg"|_##]


 Widnows XP 지원 종료로 인해서 IE 8의 점유율이 낮아지고, 자연스럽게 IE 11의 점유율이 높아질 것이라 예상해 보았습니다. 하지만 IE 8 점유율이 자연스럽게 0%에 수렴하기는 힘들 것 같고, 다양한 노력이 필요할 것으로 예상됩니다. 또한 위에서 언급한 약 17%의 IE9 사용자가 HTML5 및 CSS3 활성화에 복병이 될지도 모른다는 생각도 무시할 수 없을 것 같습니다.

 굉장히 낙관적인 예상임은 분명하지만 웹 개발자의 입장에서 사용자들이 보다 안전한 브라우저를 이용하고, 다양한 기능을 제공하기 위해서는 빠른 속도로 HTML5 와 CSS3를 폭넑게 지원하는 브라우저를 사용하거나 IE 브라우저 버전의 업그레이드가 이루어 졌으면 하는 바람입니다.
